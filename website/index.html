DNS Configuration

Point Your Domain to VOIP@ Cloud

Follow these simple steps to connect your domain to our platform

1

Choose Your Domain Type

You can use either a subdomain (e.g., pbx.yourcompany.com) or a full domain (e.g., yourcompany.com).

We recommend using a subdomain for easier management.

2

Add DNS Records

Log into your domain registrar's DNS management panel and add the following records:

Type Name/Host Value/Points To TTL
A pbx (or @ for root) YOUR_SERVER_IP 3600
SRV _sip._udp 0 5 5060 pbx.yourcompany.com 3600
NAPTR @ 10 100 "S" "SIP+D2U" "" _sip._udp 3600

The SRV and NAPTR records are optional but recommended for auto-configuration of SIP phones.

3

Wait for Propagation

DNS changes typically take 5-30 minutes to propagate, though it can take up to 48 hours in rare cases.

You can check propagation status using:

nslookup pbx.yourcompany.com
4

Access Your PBX

Once DNS has propagated, access your PBX at:

https://pbx.yourcompany.com

Log in with the admin credentials you created during registration. SSL certificate will be automatically provisioned.

Provider-Specific Guides

  1. Log in to GoDaddy.com and go to My Products
  2. Click DNS next to your domain
  3. Click Add New Record
  4. Select Type: A, Name: pbx (or @), Value: your server IP, TTL: 1 Hour
  5. Click Save
  1. Log in to Cloudflare Dashboard
  2. Select your domain and click DNS in the sidebar
  3. Click Add Record
  4. Type: A, Name: pbx, IPv4: your server IP
  5. Important: Set Proxy status to DNS Only (grey cloud icon). Do NOT proxy SIP traffic through Cloudflare.
  6. Click Save
  1. Log in to Namecheap.com and go to Domain List
  2. Click Manage next to your domain
  3. Go to Advanced DNS tab
  4. Click Add New Record
  5. Type: A Record, Host: pbx, Value: your server IP, TTL: Automatic
  6. Click the checkmark to save
  1. Go to domains.squarespace.com (Google Domains migrated here)
  2. Select your domain, then click DNS in the left menu
  3. Scroll to Custom Records
  4. Host: pbx, Type: A, Data: your server IP
  5. Click Add, then Save
  1. Open Route 53 in the AWS Console
  2. Click Hosted zones and select your domain
  3. Click Create record
  4. Record name: pbx, Record type: A
  5. Value: your server IP, TTL: 300
  6. Click Create records
  1. Go to Networking > Domains in your DO dashboard
  2. Select your domain
  3. Under Create new record, select A
  4. Hostname: pbx, Will direct to: your server IP
  5. TTL: 3600
  6. Click Create Record
